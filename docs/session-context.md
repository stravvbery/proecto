# Контекст сессии разработки — RP Messenger

## Обзор проекта

Discord-like RP мессенджер для SCP:SL комьюнити **"hondafx legacy"**. Сайт с 6 LLM-ботами, которые ведут себя как реальные люди — общаются между собой, реагируют на действия пользователя, переключаются по темам, создают слухи и диалоги.

### Фаза 3: Система ботов

**6 ботов с уникальными персонами:**

| Бот | Модель | Характер |
|-----|--------|----------|
| Hixxivxq | deepseek-v4-flash | Тихий наблюдатель, пишет коротко |
| honda | deepseek-v4-pro | Ветеран, ностальгирует по старому серверу |
| ksynaxxxxx | claude-opus-4-6 | Хаотичный мемер, рандомные темы |
| rrqxet | glm-5.1 | Параноик, видит заговоры везде |
| saishiku | claude-haiku-4-5-20251001 | Вежливый, помогает новичкам |
| vntrpz | deepseek-v4-flash | Токсичный PvP-шник, провоцирует |

**Механика ботов:**
- **Tick-система** — каждые 10-30 секунд случайный бот пишет в #болталку (когда пользователь онлайн)
- **MsgReactor** — 80% шанс что 1-3 бота ответят на сообщение пользователя
- **Chain-реакция** — 30% шанс что боты продолжат общаться между собой (до 5 цепочек)
- **EventReactor** — боты реагируют на назначение ролей, баны, анонсы
- **Память** — боты запоминают важные события и используют их в контексте
- Каждый бот получает системный промпт с персоной + историю чата + события сервера + свои "воспоминания"

### Фаза 4: Исправление багов

**Проблемы пользователя:**
> "каналы плохо сменяются, боты ничего не пишут в чат. писать сам в чат не могуу"

**Исправлено:**
1. **Переключение каналов** — route-based через Next.js router, работает стабильно
2. **Отправка сообщений** — REST POST + Socket.IO broadcast
3. **Socket-клиент** — динамическое определение URL через `/api/config` (для туннелей) и автоматический fallback на localhost
4. **Socket-хук** — асинхронное подключение с правильной очисткой эффектов
5. **TypeScript ошибка** — `socket.io.uri` (private property) заменён на ручное отслеживание `currentUrl`

### Фаза 6: Хостинг и туннели

**Попытки создать публичный доступ:**
1. **Pinggy** — SSH к `a.pinggy.io` не прошёл (сетевые ошибки с VM)
2. **LocalTunnel** — URL создавался, но нестабильный
3. **Cloudflare Tunnel** — работает с `--protocol http2`, но agentrouter.org блокирует API запросы с облачных IP

**Результат:** Приложение работает на localhost. Для хостинга нужен сервер без WAF-блокировки agentrouter.org (или альтернативный API провайдер).

## Известные ограничения

1. **AgentRouter WAF** — `agentrouter.org` использует Alibaba Cloud WAF, который блокирует запросы с облачных IP (AWS, GCP и т.д.). С домашнего IP работает нормально.
2. **Альтернативные провайдеры** — можно заменить на OpenRouter, DeepSeek, Groq (см. README)
3. **Боты не тестировались end-to-end** — из-за WAF блокировки API с VM. На localhost с рабочим API ключом должны работать.

## Структура ключевых файлов

```
server.ts                           — Socket.IO сервер, авторизация, бот-триггеры
src/lib/llm/agent-router.ts         — OpenAI клиент, модели ботов
src/lib/llm/orchestrator.ts         — Tick, MsgReactor, ChainReactor, EventReactor
src/lib/llm/personas.ts             — Персоны ботов, системные промпты, лор
src/lib/llm/memory.ts               — Система памяти ботов
src/lib/db/schema.ts                — Drizzle ORM схема (9 таблиц)
src/lib/db/seed.ts                  — Seed-данные (пользователи, каналы, роли)
src/lib/socket/client.ts            — Socket.IO клиент (runtime URL resolution)
src/hooks/useSocket.ts              — React хук для Socket.IO
src/app/app/[serverId]/[channelId]/ — Страница чата (route-based)
src/components/chat/ChatArea.tsx     — Основной компонент чата
src/components/sidebar/ChannelList.tsx — Список каналов с переключением
src/app/api/config/route.ts         — Runtime конфиг (socket URL)
```
